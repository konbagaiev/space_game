# Weapon hit / explosion FX pass

**Feature ID:** 2026-07-04-1148-weapon-hit-fx
**Worktree:** `/Users/kbagaiev/Projects/ag-wt/2026-07-04-1148-weapon-hit-fx`

## Goal

Make weapon impacts read distinctly by weapon type instead of every bullet producing the same
micro-flash, and make rocket detonations look like a real (small) explosion rather than a single
glowing blob. Two visible changes: (1) bullet hit-flash size is keyed off the weapon's `class`
— `kinetic` gets a tiny spark, `cannon` a slightly bigger (but still small) flash; (2) rocket
detonation uses a new, small/fast, layered burst (fireball layers + a few sparks + a shockwave
ring), modeled on the ship-death explosion's structure but shrunk and quick. No gameplay/balance
numbers change — this is purely FX.

## Decisions (settled — do not re-ask)

- **Q1 — bullet `class`:** Thread `class: weapon.class` onto the bullet object in `spawnBullet`.
  Add `class` to the enemy bullet weapon rows in `catalog_seed.js`: id 2 → `kinetic`, id 9 →
  `kinetic`, id 10 → `cannon`. At the hit site, `undefined`/`kinetic` → the small spark.
- **Q2 — flash sizes** (current `maxScale = 3` for all): `kinetic` → `maxScale = 0.8` (tiny spark),
  `cannon` → `maxScale = 2` (**smaller** than the current 3, a heavier-looking but still small
  flash). Color unchanged (`0xffb050`). Hardcoded `class → scale` map in the client; **no catalog
  FX data added** (DECISIONS §30).
- **Q3 — rocket burst:** Do **NOT** modify `spawnShipExplosion`. Add a **new** function
  `spawnRocketBurst(pos, blastVis, tint)` in `client/src/projectiles.js` — a small, fast layered
  burst that reuses the same pools/particle-budget gating as `spawnShipExplosion`. `detonateRocket`
  calls it instead of `spawnExplosion(pos, r.blastVis)`. Rocket-detonation **sound call unchanged**.
- **No DB schema / `db_postgres.js` change.** Weapon `class` lives inside the weapons' `stats` JSON
  blob, which is upserted from `catalog_seed.js` on every server startup (`server/src/db.js:32-34`).
  Adding `class` to three stats objects lands automatically on the next deploy restart — no migration.

## Background (verified facts the implementer can rely on)

- `weapon.class` **is** available where bullets are fired: `fireMount` (`client/src/ship-build.js:132`)
  passes the fully-resolved weapon `w` (with `class`) into `spawnBullet(muzzle, fwd, w, ...)`
  (`ship-build.js:139`). But `spawnBullet` currently stores only `damage` on the bullet, not `class`.
- Player bullet weapons already carry `class` in `catalog_seed.js`: id 1 `kinetic`, id 5 (machine gun)
  `kinetic`, id 6 (Heavy cannon) `cannon`, id 7 `kinetic`. **Enemy** bullet weapons id 2, 9, 10 have
  **no** `class` field yet.
- Adding `class` to enemy bullet weapons is **inert for sound**: `sfxFor('weapon', class, 'fire')` is
  only called for player fire (`isPlayer`-gated at `ship-build.js:137,142`), so enemy weapon fire stays
  silent regardless.
- The sim update loops for `explosions`, `sparks`, `shockwaves` (`client/src/sim.js:508-577`) are
  generic — they consume whatever `spawnExplosion` / spark / ring objects are pushed into the shared
  pools. **`spawnRocketBurst` pushes the same object shapes, so no changes to `sim.js` update loops
  are needed** — only the two spawn call sites change.

### 4. New `spawnRocketBurst` — `client/src/projectiles.js`

Add a new exported function **after** `spawnShipExplosion` (after `client/src/projectiles.js:108`),
reusing `scaledCount`, `liveParticles`, `sparkGeo`, `shockGeo`, `SPARK_COLORS`, and the shared pools.
Sizes are derived from `blastVis` (4.5 for the starter rocket, 6 for the Heavy rocket) so the burst's
footprint tracks the existing blast visual; all lifetimes are short (~0.4–0.9 s) so it reads as fast
vs. the ship burst's ~3.75 s:

```js
// ---------- Rocket detonation: a small, fast layered burst ----------
// Same structure as spawnShipExplosion (fireball layers + a few sparks + a shockwave ring) but
// shrunk and quick, so a rocket blast reads as a proper explosion rather than one glowing sphere.
// Sized off the rocket's blastVisual (R); tint is a warm rocket orange. Reuses the same particle
// pools + tier gating as the ship burst (no sim.js changes needed).
export function spawnRocketBurst(pos, blastVis = 4.5, tint = 0xffb050) {
  const R = blastVis;
  // Layered fireball: white-hot core -> tinted glow -> orange outer cloud, each bigger, slower, dimmer.
  spawnExplosion(pos, R * 0.5, 0.40, 0xffffff);                                    // white-hot core (always)
  if (G.gfx.particleScale >= 0.7) spawnExplosion(pos, R * 0.8, 0.65, tint);        // tinted glow
  spawnExplosion(pos, R * 1.15, 0.90, 0xff5a20);                                   // orange outer cloud (always)

  // A few warm sparks flung outward, clamped to the live-particle budget (like the ship burst).
  const N = Math.max(0, Math.min(scaledCount(8), G.gfx.maxParticles - liveParticles()));
  const s = R / 6; // spatial scale relative to the biggest rocket
  for (let i = 0; i < N; i++) {
    const col = SPARK_COLORS[i % SPARK_COLORS.length];
    const mat = new THREE.MeshBasicMaterial({
      color: col, transparent: true, opacity: 1,
      blending: THREE.AdditiveBlending, depthWrite: false, fog: false,
    });
    const m = new THREE.Mesh(sparkGeo, mat);
    m.position.copy(pos);
    const size = (0.2 + Math.random() * 0.3) * s;
    m.scale.setScalar(size);
    scene.add(m);
    const a = (i / N) * Math.PI * 2 + Math.random() * 0.5;
    const sp = (8 + Math.random() * 14) * s;
    const vel = new THREE.Vector3(Math.cos(a) * sp, (Math.random() - 0.5) * 4 * s, Math.sin(a) * sp);
    sparks.push({ mesh: m, vel, life: 0.5 + Math.random() * 0.6, maxLife: 1.1, size });
  }

  // Flat shockwave ring (tier-gated like the ship burst) — small + short-lived.
  if (G.gfx.particleScale >= 0.5) {
    const ringMat = new THREE.MeshBasicMaterial({
      color: tint, transparent: true, opacity: 0.9,
      blending: THREE.AdditiveBlending, depthWrite: false, fog: false, side: THREE.DoubleSide,
    });
    const ring = new THREE.Mesh(shockGeo, ringMat);
    ring.position.copy(pos); ring.position.y = 0.6;
    ring.rotation.x = -Math.PI / 2;
    scene.add(ring);
    shockwaves.push({ mesh: ring, life: 0.85, maxLife: 0.85, maxScale: R * 2.2 });
  }
}
```
